# Task: Build modules/warehouse/ (stock, movements, order.created Ôćĺ stock deduction)

## Description

Build the Warehouse module implementing the `Module` interface. Covers the stock and movements schema, API routes, and the critical event handler that subscribes to `order.created` and atomically deducts stock. If stock is insufficient for any item, the handler emits `stock.insufficient` and aborts without modifying any stock row. The module only imports from `shared/`.

## Acceptance Criteria
- [ ] `modules/warehouse/` implements the `Module` interface
- [ ] Database schema: `stock` and `stock_movements` tables with SQL migration file
- [ ] API routes: `GET /warehouse/stock`, `GET /warehouse/stock/:productId`, `GET /warehouse/movements`
- [ ] `order.created` subscriber calls `stock.deduct` for each item in a **single database transaction**
- [ ] If any item has insufficient stock: transaction is rolled back, `stock.insufficient` event is emitted, no stock row is modified
- [ ] After successful deduction: `stock_movements` row created for each item with `reason: 'order'` and `order_id`
- [ ] When `stock.qty <= stock.low_threshold` after deduction: `stock.low` event is emitted
- [ ] Frontend: `StockPage` lists all products with current stock levels (shadcn/ui `DataTable`, `Badge`, `Progress`)
- [ ] Frontend: `MovementsPage` lists stock movement history (shadcn/ui `DataTable`, `Badge`)
- [ ] All frontend components use shadcn/ui only ÔÇö https://ui.shadcn.com/
- [ ] Unit test: `order.created` with sufficient stock Ôćĺ stock row decremented, movement row created
- [ ] Unit test: `order.created` with insufficient stock Ôćĺ no rows modified, `stock.insufficient` emitted
- [ ] `tsc --noEmit` passes

## Technical Details

### Atomic deduction handler:
```typescript
// modules/warehouse/events/handlers.ts
subscribe('order.created', async (payload) => {
  await withTransaction(async (client) => {
    for (const item of payload.items) {
      const [row] = await client.query<StockRow>(
        'SELECT qty, reserved_qty, low_threshold FROM stock WHERE product_id = $1 FOR UPDATE',
        [item.productId]
      )
      if (!row || row.qty < item.qty) {
        publish('stock.insufficient', { productId: item.productId, orderId: payload.orderId })
        throw new Error('insufficient_stock')  // triggers rollback
      }
    }
    for (const item of payload.items) {
      await client.query(
        'UPDATE stock SET qty = qty - $1 WHERE product_id = $2',
        [item.qty, item.productId]
      )
      await client.query(
        'INSERT INTO stock_movements (product_id, delta, reason, order_id) VALUES ($1, $2, $3, $4)',
        [item.productId, -item.qty, 'order', payload.orderId]
      )
    }
    // check for low stock after all deductions
    for (const item of payload.items) {
      const [row] = await client.query<StockRow>(
        'SELECT qty, low_threshold FROM stock WHERE product_id = $1',
        [item.productId]
      )
      if (row && row.qty <= row.low_threshold) {
        publish('stock.low', { productId: item.productId, qty: row.qty })
      }
    }
  })
})
```

### Schema:
```sql
CREATE TABLE stock (
  product_id UUID PRIMARY KEY,
  qty INT NOT NULL DEFAULT 0 CHECK (qty >= 0),
  reserved_qty INT NOT NULL DEFAULT 0,
  low_threshold INT NOT NULL DEFAULT 5
);
CREATE TABLE stock_movements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id UUID NOT NULL,
  delta INT NOT NULL,
  reason TEXT NOT NULL CHECK (reason IN ('order','manual','restock','adjustment')),
  order_id UUID,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```
